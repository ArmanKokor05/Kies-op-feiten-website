# 📌 Definition of Done (DoD) — *Kies op feiten*

## General
- [ ] All acceptance criteria of the user story are met.
- [ ] Functionality is fully integrated into the project (frontend + backend).
- [ ] No blocking bugs or console errors remain.
- [ ] Code is pushed to the main branch and builds successfully.

---

## Frontend
- [ ] The feature works correctly in the browser with no errors or warnings.
- [ ] UI is responsive and tested on desktop, tablet, and mobile.
- [ ] Components are reusable and follow consistent structure.
- [ ] Frontend communicates successfully with the backend through API calls.
- [ ] All relevant states and edge cases are tested.

---

## Backend
- [ ] Functionality runs on Spring Boot with no errors or warnings.
- [ ] Data is correctly stored and retrieved via Hibernate.
- [ ] Endpoints are secured with JWT authentication where needed.
- [ ] Maven build runs successfully.
- [ ] Proper error handling is implemented on all endpoints.

---

## Testing & Quality
- [ ] Unit and/or integration tests cover key functionalities.
- [ ] Code is reviewed and meets team standards (code style, naming, structure).
- [ ] No console or build errors occur on frontend or backend.
- [ ] Critical functionality is manually tested.

---

## Documentation
- [ ] README contains clear steps to run the project (frontend + backend).
- [ ] API endpoints are documented (input, output, authentication).
- [ ] If applicable: a short usage guide for the feature is included.

---

## Acceptance
- [ ] The Product Owner or stakeholder has validated and approved the feature.
- [ ] The feature can be deployed to production or demo without additional work.
