# Docker Setup: Spring Boot + MySQL

This document explains how to run the **back-end Spring Boot API** and **MySQL database** using Docker. Must have [**Docker**](https://docs.docker.com/engine/install/) installed to be able to go through setup.

---

## Project Structure (Back-End)

```
backend/
├── src/                       # Application source code
├── target/                    # Compiled JAR files (generated by Maven)
├── Dockerfile                 # Dockerfile for Spring Boot
├── docker-compose.yml         # Docker Compose for app + MySQL
├── pom.xml                    # Maven build file
```

> Front-end lives in a separate folder (`frontend/`) and is not part of this Docker setup.

---

## 1. Dockerfile for Spring Boot

* Uses **Java 17 JDK**
* Copies the packaged JAR from `target/`
* Exposes port 8080
* Runs the Spring Boot app

```bash
# Build JAR first
mvn clean package

# Build Docker image
docker build -t my-springboot-app .

# Run container (optional if using Docker Compose)
docker run -p 8080:8080 my-springboot-app
```

---

## 2. Docker Compose

`docker-compose.yml` runs both **Spring Boot** and **MySQL** containers together.

**Key points:**

* Spring Boot connects to MySQL using service name `mysql-db`
* `depends_on` ensures MySQL starts first
* `mysql-data` volume persists database data

**Note**

* Only the Spring Boot application port is exposed; the MySQL container remains internal.
* Docker Compose creates a private bridge network so containers can communicate securely.
* It’s good practice **not** to expose database ports—fine for local development, but unsafe in production.


```yaml
services:
  mysql-db:
    image: mysql:latest
    container_name: mysql-db
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: ${MYSQL_DATABASE}
      MYSQL_USER: ${MYSQL_USER}
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
    volumes:
      - mysql-data:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 5s
      retries: 10
      start_period: 20s

  spring-app:
    build: .
    container_name: spring-app
    ports:
      - "8080:8080"
    environment:
      SPRING_DATASOURCE_URL: ${SPRING_DATASOURCE_URL}
      SPRING_DATASOURCE_USERNAME: ${SPRING_DATASOURCE_USERNAME}
      SPRING_DATASOURCE_PASSWORD: ${SPRING_DATASOURCE_PASSWORD}
    depends_on:
      mysql-db:
        condition: service_healthy

volumes:
  mysql-data:
```

## 3. Spring Boot Configuration

Update `application.properties`:

```properties
spring.datasource.url=jdbc:mysql://mysql-db:3306/mydb
spring.datasource.username=myuser
spring.datasource.password=mypass
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
```

---

## 4. Running the Application

1. Build JAR:

```bash
mvn clean package
```

2. Start Docker containers:

```bash
docker-compose up -d
```

3. Access API:

```
http://localhost:8080
```

4. Access MySQL CLI:

```bash
docker exec -it mysql-db mysql -umyuser -pmypass mydb
```

---

## 5. Useful Docker Commands

| Command                                                 | Purpose                           |
| ------------------------------------------------------- | --------------------------------- |
| `docker-compose up --build`                             | Build images and start containers |
| `docker-compose down`                                   | Stop and remove containers        |
| `docker logs spring-app`                                | View Spring Boot logs             |
| `docker exec -it mysql-db mysql -umyuser -pmypass mydb` | Access MySQL CLI                  |

---

## 6. Best Practices

* Store sensitive credentials in a `.env` file instead of hardcoding them see [how to use .env](./how-to-use-.env.md).
* Only expose MySQL to host in development; keep it internal in production.
* Always run `mvn clean package` before building Docker image to ensure a clean build.
